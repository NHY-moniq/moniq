# Apple App Store Review Guidelines — OnorOff 개발 참고

원본: https://developer.apple.com/app-store/review/guidelines/

OnorOff(간호사 근무표 앱) 개발 시 주의해야 할 항목을 중심으로 정리.

---

## 1. Safety

### 1.6 Data Security
- 사용자 정보의 무단 사용, 공개, 제3자 접근을 방지하는 적절한 보안 조치 구현 필수
- **OnorOff 적용**: 근무표, 팀 멤버 정보, 개인 일정 데이터 보호 → Supabase RLS + HTTPS 필수

---

## 2. Performance

### 2.1 App Completeness
- 제출 시 최종 버전이어야 함 (플레이스홀더 텍스트, 빈 웹사이트, 임시 콘텐츠 불가)
- 로그인 포함 시 **데모 계정 정보** 반드시 제공
- 데모 모드로 대체 시 앱의 전체 기능 시연 가능해야 함
- 크래시나 명백한 기술 문제가 있으면 거부
- **OnorOff 적용**: 리뷰어용 데모 관리자/멤버 계정 준비, 팀 데이터 시드 필요

### 2.3 Accurate Metadata
- **2.3.1(a)**: 숨겨진, 비활성, 문서화되지 않은 기능 금지. 모든 기능은 리뷰어가 접근 가능해야 함
- **2.3.3**: 스크린샷은 실제 앱 사용 화면이어야 함 (타이틀 아트, 로그인 화면, 스플래시만 불가)
- **2.3.5**: 가장 적절한 카테고리 선택 → **Productivity** 또는 **Business**
- **2.3.6**: 연령 등급 정직하게 응답
- **2.3.7**: 앱 이름 30자 제한, 키워드에 상표명/다른 앱 이름 사용 금지

### 2.4 Hardware Compatibility
- **2.4.1**: iPhone 앱은 가능하면 iPad에서도 동작해야 함
- **OnorOff 적용**: Flutter로 iPad/태블릿 반응형 지원

### 2.5 Software Requirements
- **2.5.1**: 공개 API만 사용, 현재 출시 OS에서 동작
- **2.5.4**: 백그라운드 서비스는 의도된 목적(VoIP, 오디오, 위치, 알림 등)에만 사용
- **2.5.5**: IPv6 전용 네트워크에서 완전 동작 필수

---

## 3. Business

### 3.1 Payments
- **3.1.1**: 앱 내 기능/콘텐츠 잠금 해제 시 **인앱 구매(IAP)** 필수. 자체 결제 메커니즘(라이센스 키, QR 등) 사용 금지
- **3.1.2 Subscriptions**: 구독 제공 시:
  - 사용자에게 지속적 가치 제공 필수
  - 구독 기간 최소 7일, 모든 기기에서 사용 가능
  - 구독 전 명확한 설명 (무엇을, 얼마에, 어떻게 취소)
  - 무료 체험은 App Store Connect에서 설정
- **OnorOff 적용**: 무료 MVP → 향후 프리미엄 기능 추가 시 IAP/구독 규칙 준수

### 3.2 Other Business Model Issues
- **3.2.2(x)**: 앱 사용을 위해 평가/리뷰/다른 앱 다운로드를 강제해서는 안 됨

## 4. Design

### 4.2 Minimum Functionality
- 앱은 단순한 웹 리패키징이 아닌, 고유한 기능/콘텐츠/UI를 포함해야 함
- **OnorOff 적용**: 네이티브 캘린더 UX, 오프라인 캐시 등으로 웹 앱과 차별화

### 4.5 Apple Sites and Services
- **4.5.4 Push Notifications**:
  - 앱 동작에 푸시 알림이 필수 조건이면 안 됨
  - 민감한 개인/기밀 정보 전송 금지
  - 프로모션/마케팅 목적 사용 시 **명시적 옵트인** 필수 + 앱 내 옵트아웃 방법 제공
  - **OnorOff 적용**: 근무표 게시 알림, 교환 요청 알림 → 사용자 동의 후 발송, 설정에서 끄기 가능

### 4.8 Login Services ★ 매우 중요
- 소셜 로그인(Google, Facebook, Kakao 등) 제공 시 **동등한 대안 로그인** 필수 제공
- 대안 로그인 요건:
  - 이름과 이메일만 수집
  - 이메일 비공개 옵션 제공
  - 광고 목적 데이터 수집 불가
- **예외**: 자사 계정 시스템만 사용하는 경우 대안 불필요
- **OnorOff 적용**:
  - 소셜 로그인 제공 시 → **Sign in with Apple** 또는 동등한 프라이버시 보호 로그인 필수 추가
  - 이메일/비밀번호만 제공하면 예외에 해당 (자사 시스템)

### 4.9 Apple Pay
- Apple Pay 사용 시 결제 정보를 사전에 제공, 브랜딩/UI 가이드라인 준수

## 5. Legal / Privacy ★ 가장 중요

### 5.1.1 Data Collection and Storage

#### (i) Privacy Policy — 필수
- **App Store Connect 메타데이터 + 앱 내**에 개인정보 처리방침 링크 필수
- 명시해야 할 사항:
  - 수집하는 데이터, 수집 방법, 모든 사용 목적
  - 제3자와 데이터 공유 시 동일한 보호 수준 보장 확인
  - 데이터 보존/삭제 정책, 동의 철회 및 삭제 요청 방법
- **OnorOff 적용**: 개인정보 처리방침 페이지 준비 필수 (이름, 이메일, 근무 일정, 팀 정보 등)

#### (ii) Permission — 동의
- 사용자/사용 데이터 수집 시 **사용자 동의** 필수 (익명 데이터라도)
- 동의 철회 방법을 쉽게 접근 가능하게 제공
- purpose string (권한 요청 문구)이 명확하고 완전해야 함

#### (iii) Data Minimization
- 핵심 기능에 필요한 데이터만 요청
- **OnorOff 적용**: 캘린더, 연락처 등 불필요한 권한 요청 금지

#### (iv) Access
- 사용자 권한 설정 존중, 불필요한 데이터 접근 유도/강제 금지
- 동의 거부 시 대안 제공 (예: 위치 거부 시 수동 입력)

#### (v) Account Sign-In ★
- 계정 기반 기능이 중요하지 않으면 로그인 없이 사용 가능해야 함
- **계정 생성 지원 시 → 앱 내 계정 삭제 기능 필수 제공**
- 앱 기능에 직접 관련되지 않는 개인 정보 입력 강제 금지
- **OnorOff 적용**:
  - 계정 삭제 기능 반드시 구현 (설정 > 계정 > 계정 삭제)
  - Supabase에서 사용자 데이터 완전 삭제 처리

### 5.1.2 Data Use and Sharing

#### (i) 데이터 사용/전송/공유 전 사용자 동의 필수
- 제3자 AI와 데이터 공유 시 명시적 공개 + 동의 필수
- App Tracking Transparency API를 통한 명시적 동의 없이 추적 금지
- 푸시 알림/위치/추적 활성화를 앱 사용 조건으로 강제 금지

#### (ii) 한 목적으로 수집한 데이터를 다른 목적으로 재사용 금지

### 5.1.5 Location Services
- 위치 서비스는 앱 기능에 직접 관련된 경우에만 사용
- 수집 전 사용자 통지 및 동의 필수

## OnorOff 앱 심사 대비 체크리스트

### 제출 전 필수 사항
- [ ] 앱 크래시/버그 테스트 완료
- [ ] 모든 메타데이터 완전하고 정확함
- [ ] **리뷰어용 데모 계정** 준비 (관리자 + 일반 멤버)
- [ ] 백엔드 서비스(Supabase) 라이브 상태
- [ ] 비명시적 기능은 App Review 노트에 상세 설명

### Privacy 필수 구현
- [ ] **개인정보 처리방침** 웹페이지 + 앱 내 링크
- [ ] **App Store Privacy Labels** (App Store Connect) 정확히 작성
- [ ] **계정 삭제** 기능 (설정 > 계정에서 접근 가능)
- [ ] 데이터 수집 동의 UI (첫 사용 시)
- [ ] 권한 요청 시 명확한 purpose string

### Login 필수 구현
- [ ] 소셜 로그인 제공 시 → **동등한 프라이버시 보호 로그인** 추가 (Sign in with Apple 권장)
- [ ] 또는 이메일/비밀번호 자체 로그인만 제공 (소셜 로그인 없이)

### Push Notification 규칙
- [ ] 알림이 앱 동작 필수 조건 아님
- [ ] 사용자 옵트인 후에만 프로모션성 알림 발송
- [ ] 앱 내 알림 끄기 옵션 (설정 > 알림)
- [ ] 민감한 개인 정보를 알림 내용에 포함하지 않음

### Design 규칙
- [ ] iPad 호환성 (Flutter 반응형)
- [ ] 앱 스크린샷은 실제 사용 화면
- [ ] 앱 이름 30자 이내
- [ ] 적절한 카테고리 선택 (Productivity/Business)

### 향후 결제 추가 시
- [ ] 인앱 구매(IAP) 사용 (자체 결제 불가)
- [ ] 구독 전 명확한 정보 제공 (가격, 기간, 취소 방법)
- [ ] 구독은 모든 기기에서 사용 가능
